# Manipulando datos por la CLI

Aunque parezca aterradora para principiantes, la Interfaz de Línea de Comandos (CLI) es una forma poderosa de tratar directamente con la máquina y los datos a analizar. Es una de las formas habituales de trabajo en Bioinformática y en ciencias de datos. Es también por la CLI que accedemos a servidores computacionales remotos para la ejecución de tareas más complejas.

## Continuando con nuevos comandos útiles en la terminal

- Vamos a trabajar con un archivo fasta de secuencias de nucleótidos. Este archivo está alojado en otro servidor; realizaremos la copia del archivo remotamente utilizando el comando `wget`. Coloque el archivo en un directorio de trabajo `~/clases`:

```
cd ~/clases
```

- Si aún no tiene este directorio/carpeta creado, créelo con el comando de abajo:

```
mkdir clases
```

- Agora executaremos o `wget`:

```
wget '[https://drive.google.com/uc?export=download&id=15OCaUzNHqp2hD6sfAnQPxebBclr6tQTS](https://drive.google.com/uc?export=download&id=15OCaUzNHqp2hD6sfAnQPxebBclr6tQTS)' -O teste.fasta
```

- Agora vamos a conocer algunos otros comandos útiles, utilizando *pipes* de diferentes comandos. Vamos a ver las últimas 10 líneas de un archivo, haciendo un *pipe* de dos comandos, el `tail` y el `head`.

```
tail -n 20 teste.fasta | head
```

> *El primero, con la opción `-n 20`, da las últimas 20 líneas del archivo, y haciendo el 'pipe' con el `head` mostrará las primeras 10 líneas de las últimas 20.*

- ¿Cuántas secuencias posee este archivo multifasta?

```
grep -c '>' teste.fasta 
```

- ¿Extraemos solo los encabezados de las secuencias fasta?

```
grep '>' teste.fasta
```

- Mostrar las líneas del archivo que comienzan con un codón de iniciación ATG:

```
grep "^ATG" teste.fasta
```

*En este caso, el carácter `^` busca patrones al inicio de una línea.*

- Ahora vamos a contar cuántas líneas en un archivo contienen los tripletes 'CAT' o 'GAT':

```
grep -c '[CG]AT' teste.fasta
```

> *La opción `-c` del comando `grep` cuenta el número de líneas. La descripción '[CG]' denota la expresión regular para la búsqueda.*

Vamos a crear otros archivos utilizando el comando `echo` y el direccionador `>` para trabajar con los próximos comandos.

- Creando un archivo con solo letras minúsculas:

```
echo 'este archivo posee solo letras minúsculas' > archivoejemplo.txt
```

- Ahora vamos a transformar las letras minúsculas de este archivo en mayúsculas, utilizando el comando `tr`:

```
cat archivoejemplo.txt | tr 'a-z' 'A-Z'
```

- Vamos a crear un archivo para utilizar el próximo comando:

```
echo 'Chr1 Chr2 Chr1 Chr3 Chr1 Chr4 Chr1 Chr5' > archivoejemplo1.txt
```

- Ahora vamos a cambiar todas las ocurrencias de **'Chr1'** a **'Chromosome 1'** y escribir el *output* para un nuevo archivo, utilizando el comando `sed`:

```
cat archivoejemplo1.txt | sed 's/Chr1/Chromosome 1/g' > archivoejemplo2.txt
```

- Vea cómo quedó el archivo:

```
more arquivoejemplo2.txt
```

Vamos ahora a descargar el archivo `min.fasta` utilizando el `wget`. Este archivo de una secuencia de nucleótidos está todo en letras minúsculas. ¿Lo transformamos en mayúsculas?

- Primero descargue el archivo `min.fasta` ejecutando el comando de abajo:

```
wget '[https://drive.google.com/uc?export=download&id=1VPtm7uJGXiMy-wLX7Wou5_v-Om3H4Zs9](https://drive.google.com/uc?export=download&id=1VPtm7uJGXiMy-wLX7Wou5_v-Om3H4Zs9)' -O min.fasta
```

- Ahora, verifique cómo está el archivo con el comando `more`:

```
more min.fasta
```

- Ahora utilice la combinación de los comandos `cat` y `tr`:

```
cat min.fasta | tr 'a-z' 'A-Z'
```

- Como no especificó ningún archivo de salida, la salida del comando fue a la pantalla. Ahora repita el comando de esta forma:

```
cat min.fasta | tr 'a-z' 'A-Z' > MAY.fasta
```

## Obteniendo datos de un genoma *online*

Vamos ahora a descargar una tabla de *features* cromosómica (información y características) de la base de datos SGD (*Saccharomyces* Genome Database). Estos archivos también pueden ser accedidos por el navegador [AQUÍ](https://drive.google.com/uc?export=download&id=1AQ1d3IGNOWFdsZkTLuC2ndJbgKGKn9yR "null").

> *El comando curl no funciona en Cocalc. Descargue el archivo por el enlace de arriba y agréguelo en CoCalc en File > New.*

Usaremos ahora otro comando para descargar archivos remotos, el comando `curl`:

```
curl [http://sgd-archive.yeastgenome.org/curation/chromosomal_feature/SGD_features.tab](http://sgd-archive.yeastgenome.org/curation/chromosomal_feature/SGD_features.tab) > sc.tab
```

> *Observe que en el comando `curl` tenemos que especificar un archivo local utilizando el redireccionamiento `>`. Si no se utiliza, el estándar es la salida a la pantalla.*

Vamos también a descargar el archivo [README](https://drive.google.com/uc?export=download&id=1-NrHJv_Ypiq2HrsAeDbECOqRZFMTG50a "null") que describe cómo está organizado el archivo de *features* de arriba:

```
curl [http://sgd-archive.yeastgenome.org/curation/chromosomal_feature/SGD_features.README](http://sgd-archive.yeastgenome.org/curation/chromosomal_feature/SGD_features.README) > README
```

Verifique qué describe cada columna del archivo **'sc.tab'**, viendo el archivo **README**:

```
more README
```

> ¡El archivo está en inglés! Acostúmbrese. Prácticamente todo en bioinformática es en lengua inglesa. Por lo tanto, en buen inglés: *Dude, get over it and move on! Get a dictionary or use Google Translator or (even better) learn english!*

## Obteniendo información sobre el archivo

El archivo **sc.tab** es un archivo con información de regiones cromosómicas de la levadura *Saccharomyces cerevisiae* donde la información referente a las columnas está separada por 'tabs'. Es un archivo con innumerables columnas y líneas, extremadamente grande. Este mismo archivo puede ser importado a una hoja de cálculo (estilo Excel®). Aun así, su visualización es complicada debido a su tamaño, además de que no siempre es posible abrir un gestor de hojas de cálculo en una terminal. Aquí usaremos una serie de comandos simples de UNIX (algunos ya vistos en el tutorial anterior, ¡háganlos por favor!) para obtener y extraer información de este archivo, sin tener que abrirlo y verificar línea a línea.

Para verificar cuántas líneas posee este archivo:

```
wc -l sc.tab
```

Para verificar cómo inicia el archivo:

```
head sc.tab
```

Para verificar qué líneas poseen el patrón deseado, por ejemplo el patrón 'YAL060W':

```
grep YAL060W sc.tab | head
```

Para colorear el patrón buscado y facilitar la visualización en la pantalla de la terminal:

```
grep YAL060W sc.tab --color=always | head
```

Podemos también extraer las líneas que contienen el patrón para un nuevo archivo:

```
grep YAL060W sc.tab > match.tab
```

El comando de abajo hace exactamente lo mismo que el anterior. Es más largo y ejecuta un programa más (`cat`), pero es más claro respecto al anterior, pues especifica mejor el archivo a abrir:

```
cat sc.tab | grep YAL060W > match1.tab
```

Para verificar cuántas líneas en el archivo poseen la palabra **'gene'**:

```
cat sc.tab | grep gene | wc -l
```

El comando de arriba encuentra la palabra **gene** en cualquier lugar de la línea. Este archivo usa la denominación *feature type* (Columna 2) 'ORF' para genes que codifican proteínas. Construyendo los comandos en pequeños pasos, extraeremos las informaciones que contienen ORF, con el comando `cut`:

```
cat sc.tab | head
cat sc.tab | cut -f 2 | head
cat sc.tab | cut -f 2 | grep ORF | head
```

> *Note que la opción* `-f 2` *especifica que queremos la columna 2 de este archivo*.

Podemos seleccionar múltiples columnas de interés a partir de la salida (*output*), también con el comando `cut`:

```
cat sc.tab | cut -f 2,3,4 | grep ORF | head
```

Podemos también obtener la información de cuántas columnas poseen la palabra **ORF** en la segunda columna:

```
cat sc.tab | cut -f 2,3,4 | grep ORF | wc -l
```

¿Y las líneas que **NO POSEEN** cierto patrón? También podemos obtenerlas utilizando la opción `-v` del comando `grep`, que invierte su acción, mostrando las líneas que no poseen el patrón. En este caso, queremos extraer las líneas que no poseen el patrón ***Dubious***:

```
cat sc.tab | grep -v Dubious | cut -f 2,5,9 | wc -l
```

Cuántas *features* existen en estos datos:

```
cat sc.tab | cut -f 2 > features.tab
```

Al clasificar, colocamos entradas idénticas consecutivas próximas unas de otras:

```
cat features.tab | sort | head 
```

Para encontrar palabras únicas, el comando `uniq` resume palabras idénticas consecutivas en solo una:

```
cat features.tab | sort | uniq | head 
```

Utilizando la opción `-c` en el comando `uniq` no solo resumirá entradas consecutivas, sino que también mostrará su conteo:

```
cat features.tab | sort | uniq -c | head 
```

No se preocupe si no sabe estos comandos y sus respectivas opciones de memoria. Esto viene con el tiempo y el uso. En un momento u otro, bioinformáticos experimentados necesitan consultar el manual (comando `man`) o recordar qué hacen estos comandos. Vale siempre dejar en los favoritos un sitio que contenga una lista con los principales comandos UNIX. Búsquedas en Google o consultas al Chat-GPT también ayudan bastante en esos momentos en que necesitamos recordar comandos.
